# Contributing

## Commits Guide

Use [conventional commit style](https://www.conventionalcommits.org/en/v1.0.0/) to push commits

### Commit Types

| Type       | Description              | Usage Examples                               |
| ---------- | ------------------------ | -------------------------------------------- |
| `feat`     | New features             | New learning module, infrastructure addition |
| `fix`      | Bug fixes                | Resolving configuration issues               |
| `docs`     | Documentation updates    | README improvements, learning notes          |
| `style`    | Code formatting          | Fixing indentation                           |
| `refactor` | Code restructuring       | Improving project organization               |
| `test`     | Test-related changes     | Adding test cases                            |
| `chore`    | Maintenance tasks        | Dependency updates                           |
| `build`    | Build system changes     | CI/CD script modifications                   |
| `ci`       | Continuous Integration   | GitHub Actions updates                       |
| `perf`     | Performance improvements | Optimizing scripts                           |

### Commit Examples

- `feat(docker): add docker compose for flink, paimon and minio`
- `fix(kubernetes): correct service port mapping`
- `docs(flink): update Flink notes`
